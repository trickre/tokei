# design.md

## 1. アプリ概要

本アプリは、Vite + TypeScript + 素のCSSで構築された単一ページのWebアプリである。
主な機能は以下の2つ。

- 現在時刻をリアルタイム表示するデジタル時計
- 指定した時間から減算するカウントダウンタイマー

加えて、時計表示エリアには全画面表示の切替機能を持たせている。
タイマー終了時には通知用の電子音を再生する。

## 2. 技術構成

- ビルドツール: Vite
- 言語: TypeScript
- スタイル: 素のCSS
- 実行環境: モダンブラウザ
- 使用ブラウザAPI:
  - `setInterval`
  - `Date`
  - `Intl.NumberFormat`
  - `FormData`
  - Fullscreen API
  - Web Audio API

外部UIライブラリや状態管理ライブラリは使用しない。
DOMを直接操作するシンプルな構成を採用する。

## 3. 画面構成

画面は大きく2つのパネルで構成する。

### 3.1 時計パネル

- 現在時刻を `HH:MM:SS` 形式で表示
- 現在日付を日本語ロケールで表示
- 全画面表示の切替ボタンを配置

### 3.2 タイマーパネル

- 分入力欄
- 秒入力欄
- タイマー全画面モード用チェックボックス
- 開始ボタン
- 一時停止 / 再開ボタン
- リセットボタン
- 残り時間表示
- タイマー状態表示
- 補助メッセージ表示
- タイマー終了時の通知音再生

## 4. レイアウト方針

- デスクトップでは2カラム構成
- モバイルでは1カラム構成
- 全体はダークトーン基調
- 時計パネルとタイマーパネルはカード状に見せる
- 全画面表示時は時計パネルのみを対象にし、時刻表示を大きく中央配置する
- タイマー開始時は設定に応じてタイマーパネルも全画面表示できる
- タイマーパネル全画面時は残り時間のみを中央に大きく表示する

## 5. ファイル構成

- `index.html`
  - アプリのエントリーポイント
- `src/main.ts`
  - UI生成
  - イベント登録
  - 時計更新処理
  - タイマー状態管理
  - 全画面切替処理
  - タイマー開始時の全画面遷移制御
- `src/style.css`

## 6. 時計機能の設計

### 6.1 表示内容

- 時刻: `HH:MM:SS`
- 日付: `ja-JP` ロケールの年月日と曜日

### 6.2 更新方式

- 初期表示時に即時更新する
- その後 `setInterval` を1秒ごとに実行し、現在時刻を更新する

### 6.3 フォーマット方針

- `Intl.NumberFormat('ja-JP')` を用いてゼロ埋め2桁表示とする
- 日付は `toLocaleDateString('ja-JP', ...)` を用いる

## 7. カウントダウンタイマー機能の設計

### 7.1 入力仕様

- 分: `0` 以上 `999` 以下
- 秒: `0` 以上 `59` 以下
- タイマー全画面モードチェックボックス: 開始時の表示モードを指定
- 合計1秒未満は無効入力とし、メッセージ表示のみ行う

### 7.2 状態管理

タイマー状態は以下の4状態で管理する。

- `idle`
- `running`
- `paused`
- `finished`

この状態に応じて、表示ステータスと一時停止ボタンの文言を切り替える。

### 7.3 管理データ

- `totalDurationMs`
  - 設定されたタイマー全体時間
- `remainingMs`
  - 現在の残り時間
- `intervalId`
  - タイマー更新用のインターバルID
- `targetTimestamp`
  - タイマー終了予定の絶対時刻

### 7.4 動作方式

開始時には残り時間そのものを減算し続けるのではなく、終了予定時刻を保持する。
更新時は `targetTimestamp - Date.now()` で残り時間を算出する。
これにより、`setInterval` の遅延による累積誤差を抑える。

### 7.5 操作仕様

- 開始
  - 入力値からミリ秒へ変換し、残り時間を初期化して開始
  - チェックボックスが有効な場合は、開始時にタイマーパネルを全画面表示する
- 一時停止
  - インターバルを停止し、状態を `paused` にする
- 再開
  - 一時停止時の `remainingMs` を基準に再度終了予定時刻を設定して開始
- リセット
  - インターバルを停止し、残り時間を設定値に戻す
- 終了
  - 残り時間が0以下になった時点で停止し、状態を `finished` にする
  - 終了時に通知音を再生する
  - タイマーパネルが全画面中であれば通常表示へ戻す

### 7.6 表示仕様

- 残り時間は `MM:SS` 形式で表示する
- 秒未満は切り上げで扱い、ユーザー視点で自然な表示にする
- 終了5秒前から残り時間表示を黄色で明滅させる
- 終了時は残り時間表示を黄色のまま2秒間維持する

## 8. 全画面機能の設計

### 8.1 対象

- 時計パネルのみを全画面表示対象とする
- タイマー開始時のオプションにより、タイマーパネルも全画面表示対象となる

### 8.2 使用API

- `requestFullscreen()`
- `document.exitFullscreen()`
- `document.fullscreenElement`
- `document.fullscreenEnabled`

### 8.3 動作

- ボタン押下時に時計パネルが未全画面なら全画面化
- すでに全画面中なら全画面を解除
- `fullscreenchange` イベントでボタン文言を同期する
- タイマー開始時にチェックボックスが有効ならタイマーパネルを全画面化する
- タイマー終了時にタイマーパネルの全画面を解除する
- タイマーパネル全画面中は、ヘッダー・入力フォーム・補助文を隠し、残り時間のみ表示する

### 8.4 例外時の扱い

- Fullscreen API が利用不可の場合は、タイマー領域のメッセージ欄にその旨を表示する

## 9. 通知音機能の設計

### 9.1 使用API

- `AudioContext`
- `OscillatorNode`
- `GainNode`

### 9.2 動作

- タイマー開始または再開時に音声コンテキストを初期化または再開する
- タイマー終了時に短い電子音を複数回再生する
- 音声ファイルは使用せず、ブラウザ上で波形を生成する

### 9.3 例外時の扱い

- Web Audio API が利用できないブラウザでは、音は鳴らさずメッセージ欄で通知する

## 10. UI生成方針

- アプリ起動時に `#app` に対して `innerHTML` で画面全体を描画する
- 各要素は `data-*` 属性で取得し、イベントと表示更新を紐付ける
- 必須要素が見つからない場合は例外を投げて初期化失敗を明示する

## 11. エラーハンドリング方針

- アプリルートや必須DOM要素が存在しない場合は `Error` を投げる
- タイマー入力が不正な場合は、処理を継続せずメッセージ欄に案内を表示する
- Fullscreen API 非対応時は処理を中断し、利用不可メッセージを表示する
- Web Audio API 非対応時は、音声再生を行わず利用不可メッセージを表示する

## 12. 非機能要件

- 軽量で依存関係が少ないこと
- ローカル開発が容易であること
- 単一ファイル中心で追いやすい構成であること
- 主要機能がモバイル幅でも利用可能であること
